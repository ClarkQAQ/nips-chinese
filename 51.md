NIP-51
======

列表
-----

`草案` `可选`

本 NIP 定义了用户可以创建的事物列表。列表可以包含对任何事物的引用，这些引用可以是**公共的**或**私有的**。

列表中的公共项目在事件 `tags` 数组中指定，而私有项目在 JSON 数组中指定，该数组模仿事件 `tags` 数组的结构，但被字符串化并使用与 [NIP-44](44_ZH.md) 相同的方案加密（共享密钥使用作者的公钥和私钥计算）并存储在 `.content` 中。此规范的早期版本使用 [NIP-04](04_ZH.md) 进行加密。这些现在已弃用。为了向后兼容，客户端可以通过在密文中搜索"iv"并相应地解密来自动发现加密是 NIP-04 还是 NIP-44。

当新项目添加到现有列表时，客户端应该将它们附加到列表末尾，以便按时间顺序存储。

## 列表类型

### 标准列表

标准列表使用普通可替换事件，意味着用户每种类型只能有一个列表。它们具有特殊含义，客户端可能依赖它们来增强用户的档案或浏览体验。

例如，_静音列表_可以包含垃圾邮件发送者和恶意行为者的公钥，用户不希望在他们的信息流中看到或收到来自这些用户的烦人通知。

| 名称              | 种类  | 描述                                                 | 预期标签项                                                                                  |
| ---               | ---   | ---                                                         | ---                                                                                                 |
| 关注列表       |     3 | 微博基本关注列表，参见 [NIP-02](02_ZH.md)        | `"p"`（公钥 -- 带有可选中继器提示和昵称）                                             |
| 静音列表         | 10000 | 用户不希望在其信息流中看到的事物          | `"p"`（公钥）、`"t"`（标签）、`"word"`（小写字符串）、`"e"`（主题）                     |
| 置顶笔记      | 10001 | 用户打算在其档案页面中展示的事件   | `"e"`（kind:1 笔记）                                                                                |

| 书签         | 10003 | 用户想要保存的未分类"全局"事物列表 | `"e"`（kind:1 笔记）、`"a"`（kind:30023 文章）、`"t"`（标签）、`"r"`（URL）                   |

| 表情符号            | 10030 | 用户首选表情符号和表情符号集指针            | `"emoji"`（参见 [NIP-30](30_ZH.md)）和 `"a"`（kind:30030 表情符号集）                                    |
| DM 中继器         | 10050 | 接收 [NIP-17](17_ZH.md) 直接消息的地方            | `"relay"`（参见 [NIP-17](17_ZH.md)）                                                                     |
| 优秀维基作者 | 10101 | [NIP-54](54_ZH.md) 用户推荐的维基作者               | `"p"`（公钥）                                                                                     |

### 集合

集合是具有明确定义含义的列表，可以增强依赖它们的客户端的功能和 UI。与标准列表不同，用户预期拥有每种类型的多个集合，因此每个集合都必须分配不同的 `"d"` 标识符。

例如，_中继器集_可以在下拉 UI 中显示，为用户提供切换到哪些中继器发布事件或从哪些中继器读取事件回复的选项；_策展集_可以被应用程序用来展示标记到不同主题的其他人制作的策展。

除了主要标识符 `"d"` 标签外，集合还可以选择性地拥有 `"title"`、`"image"` 和 `"description"` 标签，这些可以用来增强其 UI。

| 名称                  | 种类  | 描述                                                                                  | 预期标签项                                                                  |
| ---                   | ---   | ---                                                                                          | ---                                                                                 |
| 关注集           | 30000 | 客户端可能选择在不同情况下查看的分类用户组      | `"p"`（公钥）                                                                     |
| 中继器集            | 30002 | 用户在各种操作期间可以轻松选择的用户定义中继器组 | `"relay"`（中继器 URL）                                                              |
| 书签集         | 30003 | 用户定义的书签类别，用于书签必须在标记的单独组中时    | `"e"`（kind:1 笔记）、`"a"`（kind:30023 文章）、`"t"`（标签）、`"r"`（URL）   |
| 策展集         | 30004 | 用户挑选的有趣和/或属于同一类别的文章组      | `"a"`（kind:30023 文章）、`"e"`（kind:1 笔记）                                   |
| 策展集         | 30005 | 用户挑选的有趣和/或属于同一类别的视频组        | `"e"`（kind:21 视频）                                                              |
| 种类静音集        | 30007 | 按种类静音公钥<br>`"d"` 标签必须是种类字符串                                   | `"p"`（公钥）                                                                     |
| 兴趣集         | 30015 | 由一堆"标签"表示的兴趣主题                                         | `"t"`（标签）                                                                    |
| 表情符号集            | 30030 | 分类表情符号组                                                                     | `"emoji"`（参见 [NIP-30](30_ZH.md)）                                                     |
| 发布工件集 | 30063 | 软件发布的工件组                                                     | `"e"`（kind:1063 [文件元数据](94_ZH.md) 事件）、`"a"`（软件应用程序事件） |
| 应用策展集     | 30267 | 对多个软件应用程序的引用                                                 | `"a"`（软件应用程序事件）                                                  |
| 日历              | 31924 | 以任何方式分类的事件集                                                       | `"a"`（日历事件事件）                                                        |
| 入门包         | 39089 | 要共享的命名档案集，目标是一起关注         | `"p"`（公钥）                                                                     |
| 媒体入门包   | 39092 | 与上面相同，但特定于多媒体（照片、短视频）客户端                      | `"p"`（公钥）                                                                     |

### 已弃用的标准列表

一些客户端过去使用过这些列表，但它们应该努力过渡到上面的[标准格式](#标准列表)。

| 种类  | "d" 标签         | 改用                   |
| ---   | ---             | ---                           |
| 30000 | `"mute"`        | kind 10000 _静音列表_        |
| 30001 | `"pin"`         | kind 10001 _置顶列表_         |
| 30001 | `"bookmark"`    | kind 10003 _书签列表_   |
| 30001 | `"communities"` | kind 10004 _社区列表_ |

## 加密过程伪代码

```scala
val private_items = [
  ["p", "07caba282f76441955b695551c3c5c742e5b9202a3784780f8086fdcdc1da3a9"],
  ["a", "a55c15f5e41d5aebd236eca5e0142789c5385703f1a7485aa4b38d94fd18dcc4"],
]
val base64blob = nip44.encrypt(json.encode_to_string(private_items))
event.content = base64blob
```

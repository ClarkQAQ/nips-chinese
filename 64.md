NIP-64
======

国际象棋（可移植游戏记号）
------------------------------

`草案` `可选`

本 NIP 定义了 `kind:64` 笔记，表示以 [PGN][pgn_specification] 格式的国际象棋游戏，该格式可以被人类阅读，也被大多数象棋软件支持。

## 笔记

### 内容

这些笔记的 `.content` 是表示 [PGN 数据库][pgn_formal_syntax] 的字符串。

### 笔记

```jsonc
{
  "kind": 64,
  "content": "1. e4 *",
  // 其他字段...
}
```

  "tags": [
    ["alt", "费舍尔对阵斯帕斯基在贝尔格莱德 1992-11-04（F/S 回归赛，第29轮）"],
    // 其他标签...
  ],
  "content": "[Event \"F/S Return Match\"]\n[Site \"Belgrade, Serbia JUG\"]\n[Date \"1992.11.04\"]\n[Round \"29\"]\n[White \"Fischer, Robert J.\"]\n[Black \"Spassky, Boris V.\"]\n[Result \"1/2-1/2\"]\n\n1. e4 e5 2. ## 客户端行为

客户端应该将内容显示为棋盘。

客户端应该以["导出格式"][pgn_export_format]（"严格模式"，即由机器创建）发布 PGN 笔记，但期望传入的笔记采用["导入格式"][pgn_import_format]（"宽松模式"，即由人类创建）。

客户端应该检查格式是否有效，所有移动是否符合象棋规则。

客户端可能包含额外的标签（例如像 [`"alt"`](31.md)）以便向不支持的客户端的用户表示笔记。

## 中继器行为

中继器可能验证 PGN 内容并拒绝无效笔记。


## 示例

```pgn
// 一个什么都不知道的游戏。游戏仍在进行中，游戏被放弃，或结果未知。
// 也许玩家在移动之前就死了。
*
```

```pgn
1. e4 *
```

```pgn
[White "Fischer, Robert J."]
[Black "Spassky, Boris V."]

1. e4 e5 2. Nf3 Nc6 3. Bb5 {This opening is called the Ruy Lopez.} *
```
